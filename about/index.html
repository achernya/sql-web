About We're an open book

Nightly backups

Around 4 AM each day, two backups are performed: a monolithic backup to restore the entire server in the event of a catastrophe, and an individual backup for each user. At least 3 of the most recent monolithic backups are kept at all times, in addition to some historical backups that are encrypted and stored on separate servers. The nightly, individual backups are available at /mit/sql/backup/. The previous day's backup is also available upon request.

Software stack

sql.mit.edu currently runs the community edition of MySQL. We'll be upgrading to MariaDB, the newest version of the community-supported MySQL, when the servers are upgraded from Debian 6.0 (squeeze) to Debian 7.0 (wheezy). The servers run the latest version of Debathena to integrate with the MIT Athena computing environment.

All of the source code powering sql.mit.edu is open source under the FIXME license. This includes the web interface, backup scripts, and the remote control interface. The server-side code is written almost entirely in Python. The web interface is written in HTML and Javascript.

Unlike most web applications at MIT, you can't log into sql.mit.edu with a certificate. Instead, we use Webathena, a web-based implementation of Kerberos. Our servers never learn your password. In fact, your password never leaves your browser. Since the password isn't sent to the server, there's no session to log in and out of—it's all in your browser.

Server specifications

sql.mit.edu has two identical servers, named primary-key and foreign-key. Both of these servers have

  • 2x Intel Xeon E5430 (4x2.66GHz) CPUs
  • 32GB DDR2-667 ECC RAM
  • 2x2TB SATA Hard Drives in RAID1 (mirroring)
  • 2x512GB SSDs in RAID1 (mirroring)

Access and limitations

Unlike most database providers, sql.mit.edu doesn't have any restrictions on what client is used. However, since sql.mit.edu is available to the entire MIT community, there is currently a 100 megabyte quota for all accounts. If you exceed this quota, you'll be unable to make more databases, and the nightly backup will skip your account for the individual backups. Contact us to request a quota increase.
